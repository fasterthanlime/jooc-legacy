Options

Cheat sheets

Table 2-1. Cheat sheet of compiler options

-v, -verboseverbose
-g, -debugcompile with debug information
-nocleandon't delete any temporary file produced by the backend
-backend=[gcc,make,none],backendoption1,backendoption2choose the compiler backend (default=gcc)
-guisecret graphical compilation debugging tool (don't push the red button)
-daemon:portstart the compiler daemon on the specified port
-sourcepath=output/path/location of your source files
-outpathwhere to output the .c/.h files (useful with the make backend)
-I/your/include/path, -incpath=/your/include/pathwhere to find C headers
-L/your/library/path, -libpath=/your/library/pathwhere to find libraries to link with
-lmyliblink with library 'mylib'
-timingprint how much time it took to compile
-V, -version, --versiondisplay the compiler version and exit
-h, -help, --helpdisplay a short help message and exit
-r, -runruns the executable after compilation

To use backend options, simply append them to the -backend=backendname option, separated with commas (,), example:

$ ooc helloworld.ooc -backend=gcc,-g,-clean=no,-verbose=yes,-O3,-fomit-frame-pointer,-pedantic,-Wall

(a kind of extreme example, I agree)

Table 2-2. Cheat sheet of the gcc backend options

-clean=[yes,no]delete (or not) temporary files. default: yes. overriden by the global option -noclean
-verbose=[yes,no]print the gcc command lines called from the backend. overriden by the global options -v, -verbose
-shout=[yes,no], -sprints a big fat [ OK ] at the end of the compilation if it was successful (in green, on Linux platforms)
any other optionpassed to gcc

Table 2-3. Cheat sheet of the make backend options

-cc=[gcc,icl]write a Makefile to be compatible with the said compiler (only GNU gcc and Intel Compiler on Linux option styles supported yet. Easy to add, go through the source and submit a patch, if you dare)
-link=libname.alink with the static library libname.a
any other optionpassed to the compiler
